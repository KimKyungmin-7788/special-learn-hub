## 목표
현재 `/` 진입 시 로그인이 강제되어 있는데, 이를 임시로 우회합니다. 기존 로그인 페이지(`/login`)의 디자인은 그대로 유지하고, 추후 로그인 기능을 다시 활성화할 수 있도록 합니다.

## 변경 사항

### 1. `src/App.tsx` — 인증 가드 제거
- `ProtectedHome` 래퍼를 제거하고, `<Route path="/" element={<Index />} />`로 변경하여 누구나 메인 페이지에 바로 접근 가능하게 합니다.
- `Login`, `Admin` 라우트는 그대로 유지 (`/login` 페이지 자체는 디자인 보존을 위해 보존).

### 2. `src/pages/Login.tsx` — "입장하기" 버튼 추가
- 기존 좌측 그라데이션 영역 / 우측 로그인 폼 디자인은 **변경 없이 유지**합니다.
- 우측 영역 상단(또는 Google 버튼 위)에 강조된 **"입장하기"** 버튼을 하나 추가합니다. 클릭 시 `navigate("/")`로 메인 페이지로 이동.
- 안내 문구: "로그인 없이 둘러보기" 정도의 작은 보조 텍스트.
- 기존 Google / 이메일 로그인 폼은 시각적으로 유지하되, 현재는 동작하지 않을 수 있으므로 비활성화 상태로 두지 않고 그대로 둡니다 (추후 재활성화 시 손쉽게 복구 가능).

### 3. `src/components/AppHeader.tsx` (선택 사항)
- 헤더에 표시될 수 있는 로그인/로그아웃 UI는 `useAuth` 상태에 따라 자연스럽게 처리되므로 별도 수정 불필요.
- 단, "로그인" 버튼이 헤더에 노출되어 있는지 확인 후, 로그아웃 상태에서도 어색하지 않게만 유지.

## 추후 로그인 재활성화 방법 (메모용)
- `App.tsx`에서 `<Route path="/" element={<Index />} />`를 다시 `<Route path="/" element={<ProtectedHome />} />`로 되돌리면 됩니다. `ProtectedHome` 함수는 코드에서 제거하지 않고 주석 처리해두면 한 줄 변경만으로 복구 가능합니다.

## 영향 범위
- 수정 파일: `src/App.tsx`, `src/pages/Login.tsx`
- DB / 백엔드 변경 없음
- 기존 디자인/기능 보존
